Git

S.C.M. → Source Control Management.

Some Import Things

Head[Bookmark] ⇒ Git Uses the Head alias to represent the currently checked out snapshot of your project.

.gitignore ⇒ the files we want to ignore by git put here

Commit ID ⇒ The id used to tracked commits / Generated by Hash function using Sha1.

Branch ⇒ a pointer to a particular commit(Independent line of development in a project/Latest link to a chain of development history)[Default Branch MASTER].

Merge ⇒ Git Uses two different algorithms to perform a merge: Fast Forward Merge and Three-Way Merge.

Fast Forward Merge → All the commits in the checkout branch are also in a branch that being merged.

Three_Way Merged → when the history of the merging branch has diverged in some way and there is not a linear path combine them the fast forwarding.

Credential helper ⇒ save your password and username for 15min so that we don't have put our password and username every time. [Enable it by git config —global credential.helper cache]

Important Commands

Work in Git Locally

  • Tell git who we are → git config —global user.mail

    git config —global user.name

    git config -l (To check the config)

  • Start working with git → git init (make a repository)

    git clone (copy a repository)

  • Staging area → A file maintained by git that contains all of the information about what files and going to go into your next commit. (git add)
  • Status → git status
  • Commit → git commit /git commit -m ("comment for the commit").

    [what make a good commit msg → a short description of the change,followed by one or more paragraph giving more details of the change]

  • Add Commit Without Staging stage → git commit -a

    [Not work with untracked files. It works only with modified files.]

  • Add files in the same commit → git commit —amend
  • Log / Description about the commits → git log
  • Check out the changes made in our file → git log -p or git show "Commit ID"
  • Check log as graph → git graph oneline
  • To check changes in a file without committing → git diff or git add -p

    [diff —staged → to check the changes of a file which is staged but not committed]

  • Delete files from git → git rm

    [Have to commit after deleting]

  • Renaming and Moving files in Git → git mv

    [Have to commit after]

  • Revert your changes →

    git checkout [revert everything/not in staging area]

    git checkout -p [revert the file line by line]

  • Remove files from staging area → git reset [if the file is in staging area it revert]
  • Revert files from Commit → git revert HEAD [if the file is committed]

    [We can revert old files by using Commit ID git revert "Commit ID"]

Branch > "Working with branch"

  • Checking in which branch are we/list of branches → git branch
  • Creating new Branch → git branch "new branch name"
  • Move to branches → git checkout "branch name"
  • Creating new branch and at the same time move to the branch also → git checkout -b "new branch"
  • Delete branch → git branch -d "branch name"
  • Merging branches → git merge "branch name" [have to be in the main branch where we want to merge]
  • Merge Conflict → if the two branch has same file but a slight differences in commit.

[We have to use editor to fix it]

  • to abort a merge → git merge —abort

Working Git Remotely:

  • Details about the remote → git -remote -v/ git remote show origin
  • push commit in GitHub → git push
  • Accessing new updates/commits from GitHub → git pull
  • Update our branch to latest changes in remote → git fetch
  • List remote branches → git branch -r
  • fetches most up-to-date objects. → git remote update
